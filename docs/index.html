async / await as a bytecode transform

The stack frame leaves the thread.

You write an ordinary, blocking Java method. An ASM transform derives a suspending sibling: a state machine that spills its live frame into two arrays, parks on the future, and hands the thread back. No compiler plugin, no JDK 21, no class per method. Watch it happen, opcode by opcode.

02 · the trick

Most of the machinery has no reason to exist.

Scala's tree-level async phase needs an ANF transform, live-variable analysis and symbol lifting. At the bytecode level, locals are numbered slots, pattern matches are already branches, finally is already duplicated — and the operand stack is just more state to spill.

i.

The original stays byte-identical

await is a real static method whose default implementation just blocks — tier 0. Your method keeps working untransformed; the suspending sibling is derived next to it.

int x = AsyncRT.await(fa);
String s = AsyncRT.await(fb);
return s + ":" + (x * 2);
ii.

One generic frame, two arrays

No per-method field layout, no class per coroutine. Live locals and the operand stack spill into Object[] refs / long[] prims — the Quasar / Kilim trick, here with liveness-driven nulling.

199: lastore      // spill x → prims[2]
205: aconst_null
206: aastore      // fa is dead: null it
217: invokevirtual onComplete
iii.

The debugger isn't an afterthought

The LocalVariableTable is carried over and remapped — breakpoints bind in the host class, and IntelliJ shows x = 5 under its source name. Verified end-to-end over JDI.

03 · the field

This is userland Loom —
for where Loom's semantics don't fit.

A parked virtual thread can't be serialized, can't suspend against a custom scheduler, and collapses fork/join dataflow into sequential blocking. A spilled refs/prims frame is just data: persist it (durable workflows), or resume it from an actor mailbox.

Project Loom

vm-internal

Zero code changes, JDK 21+ only. Pins in synchronized; graph schedulers lose the dataflow graph.

Kotlin coroutines

compiler backend

Closest relative — but generates a typed Continuation class per coroutine. async3's generic frame needs no codegen per method.

scala-async

typed trees

The predecessor this replaces. ANF, symbol re-ownership, patmat interaction, value-class boxing — all of it disappears at bytecode level.

Quasar / Kilim

agent / aot asm

Same frame representation, unmaintained. async3 adds the runtime-triggered path and profile-driven tiering.

04 · deployment

An agent at class load.
A fallback when there isn't one.

the default — java agent

Two siblings, in your class.

At class load, every marked method gains a resumable body and an entry point bound by an LDC MethodHandle — no class generation per method. The executing bytecode lives in the class the source lines belong to, so line breakpoints just bind.

java -javaagent:async3.jar -cp … your.Main

// a constant-pool byte scan pre-filters untouched
// classes — cheap to leave on, idempotent.

Crack the lambda, transform at runtime.

Both APIs fall back to cracking the serializable lambda, reading the capturing class's bytecode back from the loader, and defining the state machine as a hidden nestmate — first use only, cached per impl method.

var sumTwice = Async.lift(Samples::sumTwice);
CompletableFuture<String> r = sumTwice.apply(fa, fb);

CompletableFuture<Integer> s = Async.async(() -> {

05 · elevation

When the blocking is one call away.

The agent finds await markers. But a method can block without one — by calling a method that awaits. indirect below has no await of its own, yet it blocks on leaf. Elevating the blocking tier gives it a suspending sibling too: every call to a suspendable method is rewritten g(…) → await(g$async(…)), and the suspension propagates up the call chain to the async boundary.

source — not an await in sight
static int indirect(CompletableFuture<Integer> f) {
    // blocks only because leaf() does
    return leaf(f) * 10;
}
derived indirect$asyncBody — the call is the suspension
invokestatic leaf$async   // → CompletableFuture
getCompleted·onComplete·tryGet  // park & resume
checkcast Integer; intValue  // Object back to int
bipush 10; imul              // …the original * 10

But when does a method get elevated? It runs blocking first, and a hot, repeatedly-blocking call witnesses its own stack — then the chain flips, bottom-up, to the async boundary. Step through the lazy tier flip:

A monotone fixpoint over the call graph finds every suspendable method (cycles and recursion converge); the blocking original is left byte-identical, and the elevated result matches it exactly — verified blocking ≡ elevated, fast path and real suspension. The within-a-class rewrite above is implemented; the witness-driven flip shown here — StackWalker sampling, on-demand hidden siblings, the MutableCallSite edge — is the runtime tier in the design doc §7.7.

Suspension with a non-empty operand stack: 1 + (2 * await(f)) + mul(await(g), 3) — the case ANF exists to forbid. It just works here.
